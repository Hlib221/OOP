Тема: Робота з класами та об'єктами 

Мета роботи: Навчитись використовувати основні принципи ООП, розглянути конструкції побудови класу та створення об'єктів і навчитись працювати з ним

Виконання роботи:
створено базову структуру програми.

Розроблено клас MyName у файлі my_class.py.

Створено скрипт main.py для демонстрації роботи класу.
Результати виконання індивідуальних завдань та відповіді на питання:

Чому створюється "Anonymous" при значенні None? У конструкторі __init__ прописана логіка: якщо переданий аргумент name дорівнює None, викликається клас-метод anonymous_user(), який присвоює ім'я "Anonymous".

Зміна привітання say_hello: Метод модифіковано. Тепер він приймає параметр message. Якщо повідомлення не передати, використовується стандартне "Hello to everyone!". У main.py продемонстровано виклик із власним текстом.

Підрахунок літер (функція з len()): У клас додано метод count_name_letters, який повертає довжину імені (len(self.name)).

Чому різна кількість імен у списку та результаті? При створенні словника (dict comprehension) використовується ім'я як ключ. Якщо у вихідному списку є однакові імена, вони перезаписують одне одного у словнику (ключі мають бути унікальні), тому кількість об'єктів у словнику може бути меншою, ніж кількість спроб створення. Однак змінна класу total_names рахує кожен виклик конструктора.

Модифікація __init__ (велика літера): Використано метод .capitalize() при ініціалізації (self.name = name.capitalize()), щоб ім'я завжди починалося з великої літери.

Зміна пошти та валідація:

Реалізовано метод change_email_domain для зміни домену.

Додано перевірку в __init__: якщо ім'я містить цифри або символи (не літери), піднімається помилка ValueError("Ім'я може містити лише літери!").

Властивість full_name: Створено @property full_name, що повертає рядок формату: User #id: name (email).

Запис у файл: Реалізовано метод save_to_file, який додає інформацію про об'єкт у файл users.txt.
